## 1. 사전 준비

### 필요한 계정
- **GitHub 계정** → [github.com](https://github.com) 에서 무료 가입
- **Netlify 계정** → [netlify.com](https://netlify.com) 에서 무료 가입 (GitHub 계정으로 가입 가능)

### 필요한 프로그램
- **Git** → [git-scm.com](https://git-scm.com) 에서 다운로드 후 설치
- **Node.js** → [nodejs.org](https://nodejs.org) 에서 LTS 버전 다운로드 후 설치

설치 확인 (명령 프롬프트 또는 터미널에서):
```
git --version
node --version
npm --version
```
각 명령어에서 버전 번호가 나오면 정상입니다.

## 2. GitHub 저장소 만들기

1. [github.com](https://github.com) 로그인
2. 오른쪽 상단 **`+`** 버튼 클릭 → **"New repository"** 클릭
3. 아래와 같이 입력:
   - **Repository name**: `ai-calendar` (원하는 이름으로 변경 가능)
   - **Description**: AI 학회 캘린더 (선택사항)
   - **Public** 선택 (Netlify 무료 플랜 사용 시 필수)
   - ⚠️ **"Add a README file" 체크 해제** (매우 중요! 체크하면 충돌 발생)
4. **"Create repository"** 클릭
5. 생성된 페이지에서 저장소 URL 확인
   - 예시: `https://github.com/내아이디/ai-calendar`

## 3. 코드를 GitHub에 올리기

명령 프롬프트(또는 터미널)를 열고 아래 명령어를 **순서대로** 실행합니다.

### 3-1. 프로젝트 폴더로 이동
```bash
cd C:\Users\frees\projects\ai-calendar
```

### 3-2. Git 최초 설정 (처음 한 번만)
```bash
git config --global user.email "내이메일@gmail.com"
git config --global user.name "내이름"
```

### 3-3. Git 저장소 초기화 및 첫 커밋
```bash
git init
git add .
git commit -m "init: AI 학회 캘린더 초기 설정"
```

### 3-4. GitHub 저장소 연결 및 업로드
```bash
git remote add origin https://github.com/내아이디/ai-calendar.git
git branch -M main
git push -u origin main
```

> 처음 push 시 브라우저에서 GitHub 로그인 창이 열릴 수 있습니다. 로그인하면 자동으로 진행됩니다.

### ✅ 확인
GitHub 저장소 페이지를 새로고침하면 파일들이 올라와 있어야 합니다.

## 4. Netlify에 사이트 배포하기

### 4-1. Netlify 가입 및 로그인
1. [app.netlify.com](https://app.netlify.com) 접속
2. **"Sign up with GitHub"** 클릭 → GitHub 계정으로 로그인
3. GitHub 권한 허용

### 4-2. 새 사이트 만들기
1. 대시보드에서 **"Add new site"** 클릭
2. **"Import an existing project"** 클릭
3. **"GitHub"** 클릭
4. 저장소 목록에서 **`ai-calendar`** 선택

### 4-3. 빌드 설정 확인
아래 설정이 자동으로 채워집니다. 그대로 두세요:
- **Build command**: `npm run build`
- **Publish directory**: `out`

**"Deploy site"** 클릭

### 4-4. 배포 완료 확인
- 초록색 체크 표시가 뜨면 배포 성공
- 사이트 주소 확인 (예: `https://aiconferences.netlify.app`)
- 주소를 클릭해서 사이트가 잘 뜨는지 확인

> 배포까지 1~2분 정도 걸립니다.

## 5. 관리자 인증 설정 (Netlify Identity)

관리자만 일정을 등록/수정/삭제할 수 있도록 인증을 설정합니다.

1. 브라우저 주소창에 아래 주소 입력:
   ```
   https://app.netlify.com/sites/내사이트명/settings/identity
   ```
   예시: `https://app.netlify.com/sites/aiconferences/settings/identity`

2. **"Enable Identity"** 버튼 클릭

3. **Registration preferences** 섹션에서:
   - **"Invite only"** 선택
   - (이렇게 해야 나만 관리자로 가입 가능)

## 6. Git Gateway 설정

Decap CMS(관리자 패널)가 GitHub에 파일을 저장할 수 있도록 연결합니다.

### 6-1. GitHub 개인 액세스 토큰 발급
1. [github.com](https://github.com) 로그인
2. 오른쪽 상단 프로필 클릭 → **"Settings"**
3. 왼쪽 메뉴 맨 아래 **"Developer settings"** 클릭
4. **"Personal access tokens"** → **"Tokens (classic)"** 클릭
5. **"Generate new token (classic)"** 클릭
6. 입력:
   - **Note**: `netlify-git-gateway`
   - **Expiration**: `No expiration`
   - **repo** 체크박스 체크 ✅ (목록 맨 위 항목)
7. 페이지 맨 아래 **"Generate token"** 클릭
8. 생성된 토큰 복사 (**⚠️ 이 페이지를 벗어나면 다시 볼 수 없음!**)

### 6-2. Netlify에 토큰 등록
1. 아래 주소 접속:
   ```
   https://app.netlify.com/sites/내사이트명/settings/identity
   ```
2. 스크롤 내려서 **"Git Gateway"** 섹션 찾기
3. **"Enable Git Gateway"** 클릭
4. **"GitHub Access Token"** 필드에 복사한 토큰 붙여넣기
5. **"Save"** 클릭

## 7. 관리자 계정 만들기

1. 위 Identity 설정 페이지 상단으로 다시 올라가기
2. **"Invite users"** 버튼 클릭
3. 본인 이메일 주소 입력 후 **"Send"** 클릭
4. 이메일 받은편지함 확인 (스팸 폴더도 확인)
5. **"Accept the invite"** 링크 클릭
6. 비밀번호 설정 후 저장

## 8. 관리자 패널 사용법

### 접속
```
https://내사이트주소/admin/
```
예시: `https://aiconferences.netlify.app/admin/`

### 일정 등록
1. 왼쪽 메뉴 **"AI 학회 일정"** 클릭
2. 오른쪽 상단 **"New 일정"** 클릭
3. 아래 항목 입력:
   - **학회명**: 캘린더에 표시될 이름
   - **시작일 / 종료일**: 날짜 선택
   - **장소**: 예) 서울 코엑스, 온라인
   - **주최기관**: 예) NeurIPS Foundation
   - **분류**: 컨퍼런스 / 워크샵 / 세미나 등
   - **기타 정보**: 논문 마감일, 주요 주제 등
   - **공식 사이트 링크**: https://로 시작하는 URL
   - **표시 색상**: 캘린더에서 보여질 색상
4. 오른쪽 상단 **"Publish"** 클릭

> 등록 후 약 1~2분 뒤 사이트에 자동 반영됩니다 (Netlify 자동 재배포).

### 일정 수정
1. **"AI 학회 일정"** 클릭
2. 수정할 일정 클릭
3. 내용 수정 후 **"Publish"** 클릭

### 일정 삭제
1. **"AI 학회 일정"** 클릭
2. 삭제할 일정 클릭
3. 상단 **"Delete"** 버튼 클릭 → 확인

## 9. 이후 코드 업데이트 방법

사이트 기능을 수정하거나 추가한 뒤 반영하는 방법입니다.

```bash
cd C:\Users\frees\projects\ai-calendar

```

push 하면 Netlify가 자동으로 감지해서 1~2분 내에 사이트를 재배포합니다.

## 10. 문제 해결

### "src refspec main does not match any" 오류
커밋이 없는 상태에서 push 시도한 경우입니다.
```bash
git add .
git commit -m "init"
git push -u origin main
```

### "rejected - fetch first" 오류
GitHub 저장소에 README 등 파일이 있어 충돌하는 경우입니다.
```bash
git pull origin main --allow-unrelated-histories
git push -u origin main
```

### "Your Git Gateway backend is not returning valid settings" 오류
`public/admin/config.yml` 에 사이트 URL이 명시되어 있는지 확인하세요:
```yaml
backend:
  name: git-gateway
  branch: main
  identity_url: https://내사이트.netlify.app/.netlify/identity
  gateway_url: https://내사이트.netlify.app/.netlify/git

site_url: https://내사이트.netlify.app
```

### 관리자 초대 메일이 안 왔을 때
- 스팸 폴더 확인
- Netlify Identity 페이지에서 다시 **"Invite users"** 클릭

### 일정 등록 후 사이트에 안 보일 때
- Netlify 대시보드 → **"Deploys"** 탭에서 배포 진행 상황 확인
- 초록색 체크가 뜰 때까지 1~2분 대기

## 구조 요약

```
GitHub 저장소 (코드 보관)
    ↕ 자동 연동
Netlify (사이트 호스팅 + 자동 배포)
    ↕
관리자 패널 (/admin)
    - Decap CMS 사용
    - Netlify Identity로 로그인
    - 일정 저장 시 GitHub에 자동 커밋
    - Netlify가 변경 감지 → 자동 재배포
```
